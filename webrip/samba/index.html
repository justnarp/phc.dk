Release Note – August 2008

Introduction

SAMBA is a windows version of the fast and powerful combined sewer planning tool used by professionals and researchers for over 20 years. The simple, well founded and well understood calculation core of the SAMBA model has only undergone minor changes compared to earlier releases. The user friendly windowed interface is not the only feature that makes this new SAMBA standout from earlier versions and similar tools. A variety of added user facilities including system editing directly in the simplified system format makes SAMBA a fast and powerful planning tool. Its functionality makes it ideal for evaluating alternative scenarios, optimising system design and performance and optimising investments in the integrated urban water environment.

Changes in SAMBA Version 2005.7:

Compared to earlier releases SAMBA Version 2005.7 is provided 'as-is' under a shareware license. In addition this distributed fixes a few minor bugs. SAMBA developments now take place within the www.WaterAspects.org environment. This release further enables network reduction from within www.WaterAspects.org (version 0.12 or later).

Changes in SAMBA Version 2005.5:

Compared to earlier releases SAMBA Version 2005.5 fixes bugs related node renaming. The reduction/setup procedure blocked in earlier versions if more than 99 nodes had long names for which short names had to be generated.

Changes in SAMBA Version 2005.4:

Compared to earlier releases SAMBA Version 2005.4 fixes bugs related to reading networks link data from MEX (Mike Urban) and MEX (Mouse). The bugs related to specific material types and scaling when no scaling factor defined as optional in the new Mike Urban MEX format.

Changes in SAMBA Version 2005.3:

Compared to earlier releases SAMBA Version 2005.3 fixes bugs related to reading networks with cross sections from MEX (Mike Urban) and UND/HGF (Mouse). A user interface bug related to the selection of which surface area model (A or B) data to use when importing from MEX files has also been fix.

Changes in SAMBA Version 2005.2:

Compared to earlier releases SAMBA Version 2005.2 fixes an interface bug related to the German and Danish language options.

Changes in SAMBA Version 2005.1:

Compared to earlier releases SAMBA Version 2005.1 is Mike Urban compatible. With this new version the full pipe network can be read from either an SWF file, UND/HGF files or from a MEX file exported from Mike Urban. To export your Mike Urban sewer netork and catchment data into a MEX file you need to select File|Import / Export ..., choose the 'Export MOUSE' format and follow Mike Urban's export instructions. Note that SAMBA only considers a subset of the sewer network definitions contained in the MEX file format (as for all other imported formats (SWF, UND and HGF)).

Changes in SAMBA Version 2004.3:

Compared to earlier releases SAMBA Version 2004.3 contains a number of minor bug fixes. Fixed bugs include the handling of long node names, memory leaks, crosssection scaling and the speed of certain tasks. A new feature is that the following three files are automatically created in the results folder when you run a simulation: reducted system overview (Model.txt), the simulation settings (Settings.txt) and the results summary (Summary.txt). Note that this release of SAMBA is NOT Mike Urban compatible. A Mike Urban compatible SAMBA is expected to be released soon.

Changes in SAMBA Version 2004.1:

This version is identical to the previous version except that it is MOUSE 2004 compatible. This version contains no new features or feature improvements due to the pending release of SAMBA built on Water Aspects TM.

Changes in SAMBA Version 2003.4:

This version is identical to the previous version except that the installation of SAMBA no longer conflicts with the license system of the other DHI software products. Known bugs include: 1. A problem can arises with long time-area curves that in their most upstream have sub-catchments with effective surface area equal to zero. Work-around: set the most upstream area to a very small size different from zero (e.g. 0.001 ha). 2. During simulation excess memory is consumed and this causes a problem when running long rainfall series on large drainage systems. Work-around: increase available hard disk space and Windows maximum permissible virtual memory and close and open SAMBA between simulations. Please see also the list of known bugs under version 2003.2.

Changes in SAMBA Version 2003.3:

This version is identical to the previous version except for the following bug fix. When importing MOUSE network data (UND file) with the "Include pipe volumes" option of the import menu checked, the volumes of conduits whose cross-section are defined as CRS XZclosed or XZopen is calculated double as big as it actually is. This can produce a non-trivial error for systems where pipe storage is a significant part of the total storage and where these pipes have been define in MOUSE using the mentioned CRS cross-section types. Users who have been working with such cases dominated by pipe storage define in this way may wish to review their results with this fixed version.

Changes in SAMBA Version 2003.2:

Version 2003.2 is compatible with the newest MOUSE 2003. It has been well tested and the contains the following known bugs. Please take particular note of the first listed bug.

Known bugs:

1. The time area curves are saved in both the SIF from where they are read on simulation and in the SDS for possible later reuse in a new project. When saving the SDS a rounding problems means that there can in certain cases be introduced an error of about 0.2-0.8% on the total sum of a time area curve. When a new import of MOUSE files is made a few of the time area curves will have a sum of 1.002 to 1.008 rather that exactly 1.000. The error is not accumulative so subsequent saves of the SDS will not increase the error. While this bug may cause quite some inconvenience in terms of lacking exact reproducibility, its effect on simulation results should be minimal. It is expected that this bug has been present also in the earlier versions of the SAMBA for windows. This bug will be corrected as soon as possible and a message will be sent to SAMBA users subscribing to the release mailing list (recommended) when a new release is available.

2. Minimum allowed time step is 300 seconds. This can give stability problems for certain control strategies.

3. When running under Windows XP SAMBA has a memory leak which means that for large systems SAMBA must be shut down after running a few simulations otherwise the simulation rate will fall drastically as all RAM is consumed. This bug does not appear to be a problem under other operating systems.

4. In the control curve graph a blue line indicates the actual volume of the controlled basin rather than that of the basin/structure which the outlet capacity is being controlled by. This has no effect on the simulation.

5. When creating a new rainfall event the date format should be entered in the operating systems local time format definition and not in the format suggested in the edit windows itself.

6. When both wastewater and surface runoff concentrations for a given pollutant are set to zero, then the resulting concentration values for those pollutants are arbitrary value with no meaning. They should in this case be zero. Once even a small concentration is defined for any one of them the simulated values are correct.

Changes in SAMBA Version 2002.1 Beta:

Version 2002.1 differs from the previous release primarily in that it is compatible with the new MOUSE 2002 file formats. Whilst being able to import and reduce networks defined in the newest MOUSE 2002 network (*.UND) and hydrological (*.HGF) data files SAMBA is still able to import data from older MOUSE file formats.

In version 2002.1 the global parameters initial loss and hydrological reduction factor are now read from the hydrological data file (*.HGF). In earlier versions these parameters were given by the user in the import menu itself.   Once imported into SAMBA these parameters can be changed in the Global Data menu. Defaults are imported and a warning is given if no values are found in the HGF file or in case the import is defined in an old format *.SWF file.

SAMBA is now also able to import surface area information that has been defined using model type B in MOUSE. This feature has not been well functioning since the last DOS version SAMBA 3.42 Classic.

Other changes involve minor bug fixes primarily in the user interface, import problems and memory management.

For your information we repeat the following from the previous release:

New features in SAMBA2001 Version 2001.1 Beta:

READING UND FILES

The main enhancement compared to SAMBA2000.27 is the ability to read MOUSE's new Urban Network Data (*.UND) files. For backward compatibility it is still possible to start a SAMBA project from a *.SWF file only or the combination of *.SWF and *HGF files.

FIXED OUTLET CAPACITIES

In MOUSE 2001 it is no longer possible to enter a fixed user defined outlet capacity for an overflow structure and this is also not a part of the *.UND format. Fixed outlet capacities therefore have to be entered using SAMBA's own node editor. When the project is saved these outlet capacities are automatically saved in a *.CAP file with the same name as the *.SIF file. These can then be re-read when re-importing and reducing the system from modified *.UND and *.HGF MOUSE files.

MOUSE'S LONG NODE NAMES

SAMBA2001 still works with short node names of no more that seven characters. A truncation strategy has therefore been implemented so that users working with long names in the MOUSE environment can import their files into SAMBA without themselves having to shorten the names. A node name such as 'N57 New Town Road' would be truncated to 'N57Ne_1' and an additional node called 'N57 New Town Avenue' would truncate to 'N57Ne_2'. Advanced users may wish create their own short versions of the node names by entering these in the automatically generated LongToShortNames.txt file.

Environment Requirements

To run SAMBA your computer requires: · Installed and licensed MOUSE with SAMBA (otherwise SAMBA will run in demo mode) · 800*600 screen resolution (minimum) or higher (recommended) · 486 processor (minimum) or faster (recommended) · 16 MB RAM (minimum) or more (recommended) · Microsoft Windows XP, 2000, 98, 95 or NT 4.0 or later · Internet Explorer 4.0 or later (to be able to run SAMBA2001Help). License is obtained via DHI Software.

Installation

To install SAMBA from the CD-ROM:

  • Close all open applications on your computer
  • Insert the CD-ROM in your CD-ROM drive
  • Select installation of MOUSE SAMBA
  • Follow the instructions that appear on your screen

You can at any time install the latest SAMBA version by:

  • Go to SAMBA Download
  • Follow the instructions to download and install from the internet

Product Invocation

Start a new project by importing system structure files (*.UND or *.SWF), the catchments file (*.HGF), waste water data (*.WWF) and SAMBA data (*.SDS) from both new and older versions of MOUSE. New projects can also be created directly from the simplified SAMBA *.SIF system/format with additional SAMBA data in the *.SDS format.
